ZAP provides post-quantum cryptographic primitives for secure agent communication.

## Overview

ZAP implements NIST FIPS 203 (ML-KEM) and FIPS 204 (ML-DSA) standards:

| Algorithm | Type | Security Level | Key Size |
|-----------|------|----------------|----------|
| ML-KEM-768 | Key Encapsulation | NIST Level 3 | 1184 bytes |
| ML-DSA-65 | Digital Signature | NIST Level 3 | 1952 bytes |
| X25519 | Key Exchange | Classical | 32 bytes |

## ML-KEM-768 Key Exchange

Post-quantum key encapsulation mechanism:

```typescript
import { PQKeyExchange, MLKEM_PUBLIC_KEY_SIZE } from '@hanzo/zap';

// Generate keypair (requires PQ library)
const alice = await PQKeyExchange.generate();
const bob = await PQKeyExchange.generate();

// Alice encapsulates to Bob's public key
const [ciphertext, aliceShared] = await alice.encapsulate(bob.publicKey);

// Bob decapsulates to get same shared secret
const bobShared = await bob.decapsulate(ciphertext);

// aliceShared === bobShared (32 bytes)
```

### From Public Key Only

```typescript
const bobPublicKey = new Uint8Array(1184); // ML-KEM public key
const kem = PQKeyExchange.fromPublicKey(bobPublicKey);

// Can only encapsulate (no decapsulation)
const [ciphertext, sharedSecret] = await kem.encapsulate(bobPublicKey);
```

## ML-DSA-65 Signatures

Post-quantum digital signatures:

```typescript
import { PQSignature, MLDSA_SIGNATURE_SIZE } from '@hanzo/zap';

// Generate keypair
const signer = await PQSignature.generate();

// Sign a message
const message = new TextEncoder().encode('Hello, ZAP!');
const signature = await signer.sign(message);

// Verify signature
const valid = await signer.verify(message, signature);
console.log('Valid:', valid); // true
```

### Verification Only

```typescript
const publicKey = new Uint8Array(1952); // ML-DSA public key
const verifier = PQSignature.fromPublicKey(publicKey);

// Can only verify (no signing)
const valid = await verifier.verify(message, signature);
```

## Hybrid Handshake

Combines X25519 (classical) with ML-KEM-768 (post-quantum) for defense-in-depth:

```typescript
import { HybridHandshake, hybridHandshake } from '@hanzo/zap';

// Quick handshake (both parties)
const [initiatorSecret, responderSecret] = await hybridHandshake();
// Both secrets are identical 32-byte keys
```

### Step-by-Step Handshake

```typescript
import { HybridHandshake } from '@hanzo/zap';

// 1. Initiator creates handshake and gets public data
const initiator = await HybridHandshake.initiate();
const initData = initiator.publicData;
// Send initData.x25519PublicKey and initData.mlkemPublicKey to responder

// 2. Responder receives and responds
const [responder, respData] = await HybridHandshake.respond(initData);
// Send respData.x25519PublicKey and respData.mlkemCiphertext to initiator

// 3. Initiator finalizes to get shared secret
const initiatorSecret = await initiator.finalize(respData);

// 4. Responder completes to get same shared secret
const responderSecret = await responder.complete(initData);

// Both secrets are identical 32-byte keys derived via HKDF-SHA256
```

## Web Crypto Integration

ZAP uses the Web Crypto API for classical cryptography:

### X25519 Key Exchange

```typescript
// Available in Node.js 18+ and modern browsers
const keyPair = await crypto.subtle.generateKey(
  { name: 'X25519' },
  true,
  ['deriveBits']
);

const publicKey = await crypto.subtle.exportKey('raw', keyPair.publicKey);
```

### HKDF Key Derivation

```typescript
const ikm = new Uint8Array(32); // Input key material
const salt = new TextEncoder().encode('ZAP-HYBRID-HANDSHAKE-v1');
const info = new TextEncoder().encode('shared-secret');

const ikmKey = await crypto.subtle.importKey(
  'raw',
  ikm,
  { name: 'HKDF' },
  false,
  ['deriveBits']
);

const derived = await crypto.subtle.deriveBits(
  { name: 'HKDF', hash: 'SHA-256', salt, info },
  ikmKey,
  256
);
```

## Error Handling

```typescript
import { CryptoError, isPQAvailable } from '@hanzo/zap';

try {
  if (!isPQAvailable()) {
    throw new CryptoError('PQ crypto requires liboqs-node');
  }

  const signer = await PQSignature.generate();
} catch (error) {
  if (error instanceof CryptoError) {
    console.error('Crypto error:', error.message);
  }
}
```

## Security Notes

1. **Hybrid Approach**: Even if one algorithm is broken, the other provides protection
2. **NIST Level 3**: Equivalent to AES-192 security
3. **Forward Secrecy**: Each handshake generates new ephemeral keys
4. **Key Derivation**: HKDF-SHA256 combines both shared secrets

> Note: Full post-quantum support requires native bindings (liboqs-node). Web Crypto API provides X25519 and HKDF in all modern environments.
