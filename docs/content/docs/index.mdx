**ZAP** (Zero-Copy App Proto) is a high-performance TypeScript library for AI agent communication using ZAP RPC.

## Features

- **Zero-Copy Serialization**: ZAP for minimal memory overhead
- **Multiple Transports**: WebSocket, TCP, Unix socket, stdio
- **MCP Compatible**: Works with Model Context Protocol servers
- **Post-Quantum Ready**: ML-DSA-65 and ML-KEM-768 cryptography support
- **DID Support**: W3C Decentralized Identifiers (did:lux, did:key, did:web)
- **Browser & Node.js**: Unified API across environments

## Quick Example

```typescript
import { Client } from '@hanzo/zap';

// Connect to a ZAP gateway
const client = await Client.connect('zap://localhost:9999');

// List available tools
const tools = await client.listTools();

// Call a tool
const result = await client.callTool('search', { query: 'hello' });

// Clean up
await client.close();
```

## Architecture

ZAP operates in three modes:

1. **Client Mode**: Connect to existing MCP servers via Gateway
2. **Server Mode**: Expose tools/resources as an MCP server
3. **Gateway Mode**: Aggregate multiple MCP servers behind a single endpoint

## Supported Transports

| Transport | Browser | Node.js | Description |
|-----------|---------|---------|-------------|
| WebSocket | Yes | Yes | Browser-compatible bidirectional |
| TCP | No | Yes | Direct TCP socket connection |
| Unix | No | Yes | Unix domain socket |
| stdio | No | Yes | Standard input/output pipes |
| zap | Yes | Yes | Native ZAP protocol |
