ZAP works in modern browsers using WebSocket transport for bidirectional communication with ZAP gateways.

## Browser Compatibility

ZAP requires browsers with support for:

- **WebSocket**: All modern browsers
- **Web Crypto API**: For cryptographic operations
- **ES2022**: Modern JavaScript features

| Browser | Minimum Version |
|---------|-----------------|
| Chrome | 89+ |
| Firefox | 90+ |
| Safari | 15+ |
| Edge | 89+ |

## Basic Usage

```typescript
import { Client } from '@hanzo/zap';

// Connect via WebSocket (secure)
const client = await Client.connect('wss://gateway.example.com:9999');

// Or non-secure for development
const devClient = await Client.connect('ws://localhost:9999');
```

## React Integration

```tsx
import { useState, useEffect } from 'react';
import { Client, Tool, Resource } from '@hanzo/zap';

function useZapClient(url: string) {
  const [client, setClient] = useState<Client | null>(null);
  const [tools, setTools] = useState<Tool[]>([]);
  const [resources, setResources] = useState<Resource[]>([]);
  const [error, setError] = useState<Error | null>(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    let mounted = true;
    let zapClient: Client | null = null;

    async function connect() {
      try {
        zapClient = await Client.connect(url);
        if (!mounted) {
          await zapClient.close();
          return;
        }

        setClient(zapClient);

        const [fetchedTools, fetchedResources] = await Promise.all([
          zapClient.listTools(),
          zapClient.listResources(),
        ]);

        if (mounted) {
          setTools(fetchedTools);
          setResources(fetchedResources);
          setLoading(false);
        }
      } catch (err) {
        if (mounted) {
          setError(err as Error);
          setLoading(false);
        }
      }
    }

    connect();

    return () => {
      mounted = false;
      zapClient?.close();
    };
  }, [url]);

  return { client, tools, resources, error, loading };
}

// Usage in component
function AgentPanel() {
  const { client, tools, loading, error } = useZapClient('wss://gateway.example.com');

  if (loading) return <div>Connecting...</div>;
  if (error) return <div>Error: {error.message}</div>;

  return (
    <div>
      <h2>Available Tools</h2>
      <ul>
        {tools.map(tool => (
          <li key={tool.name}>
            <strong>{tool.name}</strong>: {tool.description}
          </li>
        ))}
      </ul>
    </div>
  );
}
```

## Tool Execution Hook

```tsx
import { useState, useCallback } from 'react';
import { Client, ZapError } from '@hanzo/zap';

function useToolExecution(client: Client | null) {
  const [result, setResult] = useState<unknown>(null);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState<ZapError | null>(null);

  const execute = useCallback(async (
    toolName: string,
    args: Record<string, unknown>
  ) => {
    if (!client) {
      setError(new ZapError('Client not connected'));
      return;
    }

    setLoading(true);
    setError(null);

    try {
      const output = await client.callTool(toolName, args);
      setResult(output);
    } catch (err) {
      setError(err as ZapError);
    } finally {
      setLoading(false);
    }
  }, [client]);

  return { execute, result, loading, error };
}
```

## Security Considerations

### CORS

The ZAP gateway must send appropriate CORS headers:

```
Access-Control-Allow-Origin: https://your-app.com
Access-Control-Allow-Methods: GET, POST, OPTIONS
Access-Control-Allow-Headers: Content-Type
```

### Secure Connections

Always use `wss://` in production:

```typescript
const client = await Client.connect(
  process.env.NODE_ENV === 'production'
    ? 'wss://gateway.example.com:9999'
    : 'ws://localhost:9999'
);
```

### Content Security Policy

Allow WebSocket connections in your CSP:

```html
<meta http-equiv="Content-Security-Policy"
      content="connect-src 'self' wss://gateway.example.com;">
```

## Web Crypto API

ZAP uses the Web Crypto API for cryptographic operations available in all modern browsers:

```typescript
import { isWebCryptoAvailable } from '@hanzo/zap';

if (isWebCryptoAvailable()) {
  console.log('Web Crypto available');
  // X25519, HKDF, SHA-256 all work
}
```

Features available via Web Crypto:

- X25519 key exchange
- HKDF key derivation
- SHA-256 hashing
- Random number generation

> Note: Full post-quantum cryptography (ML-DSA, ML-KEM) requires native bindings not available in browsers.

## Offline Support

For offline-first applications, cache server capabilities:

```typescript
const CACHE_KEY = 'zap-capabilities';

async function getCapabilities(client: Client) {
  const cached = localStorage.getItem(CACHE_KEY);

  if (cached) {
    return JSON.parse(cached);
  }

  const [tools, resources] = await Promise.all([
    client.listTools(),
    client.listResources(),
  ]);

  const capabilities = { tools, resources };
  localStorage.setItem(CACHE_KEY, JSON.stringify(capabilities));

  return capabilities;
}
```
